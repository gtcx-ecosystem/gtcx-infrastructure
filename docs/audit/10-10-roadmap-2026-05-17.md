```
Week 1–2 (M1: Cap Lift):
  → Enable Vault TLS + fix container contexts (serial: cluster restart)
  → Send pen-test RFP (parallel, external)
  → Target: Core 5.9 → 7.0 (cap lifted)

Week 3–8 (M2: Hardening):
  → Parallel engineering:
    → Coverage improvements, eval pipeline, model cards, low-bandwidth
    → Cross-repo package adoption, NetworkPolicy patches, ALB scoping
    → ZAP workflow consolidation
  → Parallel external:
    → Pen-test vendor negotiation, SOC 2 auditor kickoff
  → Target: Core 7.0 → 8.0

Week 9–22 (M3: Certification):
  → Pen-test execution + remediation
  → SOC 2 gap analysis + evidence collection
  → mTLS mesh phased rollout
  → Anomaly detection production deploy
  → Target: Core 8.0 → 9.0

Week 23+ (M4: Reference):
  → SOC 2 Type 1 attestation
  → Red team exercise + bug bounty
  → All external evidence consolidated
  → Target: Core 9.0 → 10.0
```

**Critical path duration:** 6–9 months
**Parallelizable work:** All M2 engineering items run in parallel. Vault TLS + container fixes are the only serial M1 items.
**Longest external dependency:** SOC 2 Type 1 (3–6 months from auditor engagement)
