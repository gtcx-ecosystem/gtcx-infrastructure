# 6 — Testing

> Test plans, QA documentation, and UAT results.

---

## What Belongs Here

- Test plans (per feature or per sprint)
- QA checklists and sign-off records
- UAT results and acceptance records
- Bug reports linked to specific sprints or epics
- Performance and load test results

Name files by sprint or feature: `sprint-{N}-test-plan.md` or `{feature}-uat.md`

## Templates

| File                                         | Description                                                                          |
| -------------------------------------------- | ------------------------------------------------------------------------------------ |
| [qa-test-plan.md](qa-test-plan.md)           | QA test plan template — scope, strategy, environments, test cases, defect management |
| [uat/uat-test-plan.md](uat/uat-test-plan.md) | UAT test plan template — business scenarios, team roles, issue management, sign-off  |
