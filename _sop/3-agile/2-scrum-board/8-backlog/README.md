# Backlog

Prioritized product backlog using MoSCoW method.

## Contents

| File                                               | Description                                                                                        |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| [backlog.md](backlog.md)                           | Backlog template — Must Have, Should Have, Could Have, Won't Have; organized by platform and phase |
| [feedback-improvement.md](feedback-improvement.md) | Feedback and continuous improvement — measurement, channels, experimentation, KPIs, review cadence |

## What belongs here

- MoSCoW prioritized backlog documents
- Feature requests awaiting sprint assignment
- Won't-have decisions and rationale
- Backlog grooming outputs

## What does NOT belong here

- Active sprint items (→ `../5-sprints/`)
- Phased roadmaps (→ `../2-phases/`)
- Retrospective action items (→ `../7-retrospectives/`)
