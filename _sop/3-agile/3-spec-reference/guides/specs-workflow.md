# Guide: Specs Workflow

How to create and maintain specifications consistently across products and services.

## When to Write a Spec

- New product or service is initiated
- Major feature with cross-team dependencies
- API contract changes or new integrations

## Workflow

1. **Select the spec type**
   - Product spec, technical spec, or user story
2. **Use the template**
   - Start in `../templates/`
3. **Place the spec**
   - Product repos: `docs/specs/{group}/{service}/`
4. **Review for completeness**
   - Scope, constraints, success criteria, risks
5. **Publish and link**
   - Ensure it is linked from the nearest README

## Quality Bar

- Clear objective and success criteria
- Explicit scope and non-goals
- Concrete acceptance criteria
- Dependencies and risks documented
