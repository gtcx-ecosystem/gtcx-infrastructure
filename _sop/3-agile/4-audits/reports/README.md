# Audit Reports

Completed audit reports live here. Use the audit templates in `../templates/` to create new reports.

## Contents

Reports are saved per audit cycle using the naming pattern `{YYYY-MM-DD}-{scope}-audit-report.md`.

## What Belongs Here

- Completed repository hygiene audit reports
- Completed documentation audit reports
- Completed accessibility audit reports
- Completed security audit reports
- Any audit output generated from templates in `../templates/`

## What Does NOT Belong Here

- Audit templates — those live in `../templates/`
- In-progress or draft audits
- Retrospective notes — those live in `../../2-scrum-board/7-retrospectives/`
