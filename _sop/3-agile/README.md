# \_sop/3-agile — Agile Planning

Sprint planning, roadmap tracking, backlog management, and UAT evidence for `gtcx-protocols`.

## Contents

| Path                                             | Description                                                    |
| ------------------------------------------------ | -------------------------------------------------------------- |
| [`roadmap.md`](roadmap.md)                       | Delivery roadmap — phases, milestones, current state           |
| [`backlog.md`](backlog.md)                       | Prioritized backlog — all items with status                    |
| [`definition-of-done.md`](definition-of-done.md) | Acceptance criteria that must be met before any item is closed |
| [`sprints/`](sprints/)                           | Per-sprint notes and review files                              |
| [`uat/`](uat/)                                   | UAT plan and execution evidence                                |

## What Belongs Here

- Roadmap updates when delivery state changes
- Sprint notes saved as `sprints/sprint-{n}-{YYYY-MM-DD}.md`
- UAT execution records under `uat/`
- Backlog additions or status updates
- Definition-of-done changes (require human approval)

## What Does NOT Belong Here

- Protocol specs, architecture, or engineering decisions — those live in `_sop/2-docs/`
- Session notes or scratch work — those live in `_sop/4-sessions/`
- Release notes or changelogs — those live in `_sop/5-release/`
- Metrics dashboards — those live in `_sop/6-metrics/`

## Current State

v3.0 delivery is complete. All 45 backlog items are closed. The repo is in ongoing maintenance mode. New work items should be added to `backlog.md` and tracked through sprints.
