# sprints/

Sprint notes for `gtcx-protocols`.

## File Naming

```
sprint-{n}-{YYYY-MM-DD}.md
```

Where `{n}` is the sequential sprint number and `{YYYY-MM-DD}` is the sprint start date.

Example: `sprint-7-2026-03-21.md`

## What Goes in a Sprint File

Each sprint file covers:

- **Sprint goal** — one sentence stating what the sprint sets out to accomplish
- **Exit criteria** — explicit, testable conditions that define done for the sprint
- **Items in scope** — backlog IDs from `../backlog.md`
- **Status at close** — what was delivered, what was not, and why
- **Velocity** — points committed vs. completed
- **Carry-over** — items that did not complete, with reason

Use the template in `_archive/legacy/legacy-SOPs/templates/reports/sprint-review.md` as a starting format.
