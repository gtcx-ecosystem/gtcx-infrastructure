# Guide: Proposal Workflow

How to create, review, and approve design proposals and RFCs.

## When to Use

- New architecture or platform decisions
- Changes that affect multiple services
- Security or compliance-impacting changes

## Steps

1. **Select template**
   - Use `../templates/`
2. **Draft the proposal**
   - Focus on objective, options, and trade-offs
3. **Review**
   - At least one technical reviewer
   - Stakeholder review if user impact is significant
4. **Decision**
   - Mark as Accepted/Rejected and record rationale
5. **Publish**
   - Link from the relevant README
