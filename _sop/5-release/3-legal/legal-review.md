## Review Triggers

Legal review is required before any release that includes:

- New data collection, storage, or processing
- Changes to terms of service or privacy policy
- New third-party integrations or data processors
- New jurisdictions or user populations
- Changes to payment, subscription, or pricing terms
- AI-generated output presented to end users
- New biometric, health, or sensitive data handling

---

## Checklist

### Terms of Service

- [ ] ToS reviewed and reflects current product scope
- [ ] ToS accurately describes user obligations and rights
- [ ] Limitation of liability clause reviewed
- [ ] Dispute resolution clause reviewed
- [ ] ToS version updated and effective date set
- [ ] Users will be notified of changes per policy

### Privacy Policy

- [ ] Privacy policy reflects all data collected in this release
- [ ] Data retention periods are accurate
- [ ] Third-party data processors listed and up to date
- [ ] User rights (access, deletion, portability) accurately described
- [ ] Cookie/tracking disclosures updated if applicable
- [ ] GDPR / CCPA / applicable jurisdiction compliance confirmed

### Data Processing Agreements

- [ ] All new third-party data processors have signed DPAs
- [ ] Existing DPAs reviewed for scope coverage
- [ ] Sub-processor list updated

### Intellectual Property

- [ ] All new third-party libraries reviewed for license compatibility
- [ ] Open-source obligations (attribution, disclosure) identified and met
- [ ] No GPL/copyleft code in proprietary product paths
- [ ] IP ownership for AI-generated outputs reviewed

### AI-Specific (if applicable)

- [ ] AI output disclosures reviewed (users informed where AI generates content)
- [ ] Bias and fairness considerations documented
- [ ] Model provider terms reviewed for use case compatibility
- [ ] Training data provenance reviewed for legal exposure

### Regulatory

- [ ] Applicable regulations identified for all jurisdictions in scope
- [ ] Compliance requirements validated against release scope
- [ ] Any new regulatory filings or notifications required
- [ ] Incident/breach notification requirements reviewed
