# Product Specs

Per-product specification documents covering requirements, scope, and acceptance criteria.

## Contents

| File                                 | Description                                                                                                  |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------ |
| [`prd-template.md`](prd-template.md) | PRD template — problem, goals, personas, requirements, user stories, rollout plan, success metrics, sign-off |
| [`user-guide.md`](user-guide.md)     | User guide template — getting started, features, workflows, troubleshooting, FAQ                             |

## What belongs here

- Product requirements documents (PRDs)
- Feature specifications with acceptance criteria
- Product scope and boundaries
- Stakeholder sign-off artifacts
- Feature flag and rollout plans

## What does NOT belong here

- Design specs (→ `2-design/`)
- Frontend architecture (→ `3-frontend/`)
- Backend architecture (→ `4-backend/`)
- Test plans (→ `6-testing/`)
