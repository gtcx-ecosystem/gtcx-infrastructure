# Frontend Specs

Frontend architecture specifications and implementation standards.

## Contents

| File                                                   | Description                                                                                              |
| ------------------------------------------------------ | -------------------------------------------------------------------------------------------------------- |
| [`frontend-architecture.md`](frontend-architecture.md) | Framework, state management, routing, build tooling, and performance targets                             |
| [`frontend-strategy.md`](frontend-strategy.md)         | Frontend strategy — design philosophy, audiences, tech stack, implementation phases, metrics, governance |
| [`frontend-prd.md`](frontend-prd.md)                   | Frontend PRD — features, performance, accessibility, security, scalability, UX requirements, approvals   |
| [`frontend-epics.md`](frontend-epics.md)               | Frontend epics — suggested epic categories with features and feature breakdown template                  |
| [`screen-inventory.md`](screen-inventory.md)           | Screen inventory and navigation map — screen list, access matrix, navigation, transitions, breakpoints   |

## What belongs here

- Frontend architecture decisions for a specific product
- Technology stack choices and rationale
- Application structure and folder conventions
- Authentication flow implementation
- Real-time update strategy
- Performance budgets and accessibility requirements

## What does NOT belong here

- Design specs and component definitions (→ `2-design/`)
- Backend API contracts (→ `4-backend/`)
- Test strategy (→ `6-testing/`)
