## Guide Structure

Each guide follows this format:

1. **Prerequisites** — What you need before starting
2. **Overview** — What you'll accomplish
3. **Steps** — Numbered, actionable instructions
4. **Verification** — How to confirm success
5. **Troubleshooting** — Common issues and solutions
6. **Next Steps** — Where to go from here

## Contributing Guides

When writing a new guide:

1. **Start with the outcome** — What will the reader achieve?
2. **Assume minimal context** — Don't assume prior knowledge
3. **Test every step** — Run through the guide yourself
4. **Include code examples** — Working code, not pseudocode
5. **Add troubleshooting** — Anticipate where people get stuck
